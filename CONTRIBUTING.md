# Como contribuir

Obrigado por considerar contribuir! Mantenha as contribuições objetivas e pequenas.

## Fluxo básico

1. Abra uma issue descrevendo o problema/feature.
2. Faça um fork e crie uma branch descritiva.
3. Rode `npm install` e `npm run build` localmente.
4. Garanta que a extensão carrega e funciona o caso proposto.
5. Abra um PR vinculando a issue e descrevendo mudanças e impacto.

## Diretrizes

- Código claro, nomes descritivos, sem comentários supérfluos.
- Evite dependências desnecessárias.
- Explique decisões não óbvias no PR.

## Padrões

- Node 18+, TypeScript 5, Vite 5.
- Não commitar `dist/`.
- Use `The Unlicense` para novos arquivos.
